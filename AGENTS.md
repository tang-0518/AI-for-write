# Repo Working Agreement

## 计划与更新记录

所有功能计划、待开发事项、版本历史统一维护在 **`PLAN.md`**。

每次完成有意义的代码改动后，在同一轮次更新 `PLAN.md`：
- 将新完成的功能移入「已完成功能」表格
- 将对应待开发条目删除
- 在「版本历史」顶部追加一条记录，包含：版本号、日期、改动摘要、关键文件
- 不等用户提醒，主动更新

如果没有文件改动，不添加记录。

## 其他文档

| 文件 | 用途 |
|------|------|
| `ARCHITECTURE.md` | 技术架构、目录结构、数据模型（技术参考，改动目录结构时同步更新）|
| `README.md` | 面向用户的项目介绍 |
| `walkthrough.md` | 功能使用指南 |
| `项目导航.md` | 项目导航 |
