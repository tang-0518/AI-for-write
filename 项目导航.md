## 一、只需要关心这 6 个核心文件

63个文件里，真正实现核心功能的只有这6个。
其他都是围绕它们的辅助。

```
src/
├── types.ts                    ← [1] 所有数据的"定义书"，先看这个
├── api/gemini.ts               ← [2] 所有 AI 功能的实现（续写/润色/摘要）
├── api/contextCompression.ts   ← [3] 长文压缩算法（正文太长时自动裁剪再发给AI）
├── hooks/useEditor.ts          ← [4] 编辑器的大脑（续写/润色的状态流转）
├── hooks/useMemory.ts          ← [5] 记忆宫殿（角色/设定/章节摘要的增删查）
└── App.tsx                     ← [6] 总装配层（把所有模块组装在一起）
```

## 二、核心功能在哪里实现

### ✍️ AI 续写
```
用户点击"续写"
  → CommandBar.tsx          触发点击事件
  → App.tsx                 转发给 useEditor
  → useEditor.ts            continueWriting()  ← 主逻辑在这里
      → contextCompression.ts  判断是否需要压缩正文
      → useMemory.ts           检索相关记忆
      → gemini.ts              streamContinue() 发送请求、流式接收
  → Editor.tsx              渐进显示输出文字
  → 用户点"接受" → 追加到正文
```

### 🪄 AI 润色
```
用户选中文字 → 点润色
  → CommandBar.tsx
  → useEditor.ts            polish()  ← 主逻辑在这里
      → gemini.ts           polishText()
          短文(≤2000字)  → polishChunk()    单次请求
          长文(>2000字)  → 分段 → 串行请求 → 拼接
  → 用户点"接受" → 替换原文选区
```

### 🧠 记忆宫殿
```
自动写入：
  用户接受续写 → App.tsx → gemini.ts extractEntitiesFromAccepted()
                         → useMemory.ts upsertExtracted() → IndexedDB

手动操作：
  MemoryPanel.tsx → useMemory.ts → memory/storage.ts → IndexedDB

注入续写：
  useEditor.ts continueWriting() → useMemory.ts buildContextForQuery()
  → 相关记忆作为背景发给 AI
```

### 📚 书目与章节
```
useBooks.ts    ← 书目列表、章节列表、切换章节
useDrafts.ts   ← 章节内容的读写（IndexedDB）
Sidebar.tsx    ← 显示书目/章节列表的 UI
db/index.ts    ← 最底层的 IndexedDB 读写操作
```

## 三、数据存在哪里

```
IndexedDB（本地数据库，浏览器内）
  ├── books          书目列表
  ├── drafts         章节内容（含正文、压缩状态）
  ├── memories       记忆宫殿条目
  ├── style_profiles 文风档案
  ├── plot_hooks     情节钩子
  └── kv             杂项键值（激活的书目ID、章节顺序等）

localStorage（浏览器缓存）
  └── novel-ai-cache-v1   AI生成结果的缓存（避免重复请求）

内存（刷新后消失）
  └── React State     当前编辑内容、UI状态、待审核的续写/润色
```

## 五、看代码的建议顺序

```
第1步  types.ts              → 先搞懂有哪些数据类型
第2步  db/index.ts           → 数据怎么存进去、读出来
第3步  memory/types.ts       → 记忆宫殿的数据结构
第4步  api/cache.ts          → 缓存是怎么工作的
第5步  api/contextCompression.ts → 长文怎么压缩
第6步  api/gemini.ts         → AI请求怎么发、流式怎么处理
第7步  hooks/useMemory.ts    → 记忆的增删查改
第8步  hooks/useEditor.ts    → 续写/润色的完整状态机
第9步  App.tsx               → 所有模块怎么组装在一起
```

## 六、遇到问题时去哪找

| 问题 | 去看哪里 |
|------|---------|
| AI续写没反应 | `useEditor.ts` continueWriting() |
| 润色结果不对 | `gemini.ts` buildPolishSystemInstruction() |
| 记忆没注入续写 | `useMemory.ts` buildContextForQuery() |
| 章节切换出bug | `useBooks.ts` + `App.tsx` |
| 数据没保存 | `db/index.ts` + `hooks/useDrafts.ts` |
| 缓存命中异常 | `api/cache.ts` makeCacheKey() |
| 正文被截断压缩 | `api/contextCompression.ts` maybeCompactForContinue() |
